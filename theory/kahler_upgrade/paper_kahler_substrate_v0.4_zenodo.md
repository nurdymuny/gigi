## Abstract

This paper extends the discrete section-graph runtime of the Davis framework with an eight-layer Kähler-geometric substrate generated by the tuple

$$\mathcal{G} = (M, g, J, \nabla, B, \Gamma)$$

— a Kähler manifold with closed magnetic 2-form and a discrete graph approximation. The layers cover dual adjacency commutativity, B-perturbed magnetic transport, Jacobi-field cardinality bounds, holomorphic curvature decomposition, Hadamard substructure detection, Hodge complex with Morse compression, and a prequantization / quantum-cohomology / Berezin–Toeplitz triple.

The layers ship behind a single feature flag with strict additivity: the no-feature build is **bit-identical** to pre-upgrade across 720 passing tests, and the feature-enabled build adds ~180 tests with zero breaking changes.

We report cross-team validation evidence from two independent consumers of the same substrate:

1. **Marcella** — a generative model whose 30-prompt A/B harness produces perfect monotonicity between residue movement and reply change (21/21 plus 9/9 across four ablation cells) and a peak per-turn non-associativity of **0.0747**, matching the closed-form L7.5 *predicted upper bound* of 7.6 percentage points to within sampling noise (gap 0.0013, below 1σ).

2. **The DPU** (Davis Geometric Processor, U.S. Provisional No. 64/073,981) — a graphene-based geometric processor whose simulated bilayer Berry phase agrees with the closed-form McCann–Koshino formula at relative error ≤ 3×10⁻⁴ across a seven-point bias sweep, and whose mid-bias Dirac-string deviation matches the catalog's toy Wu–Yang reference at four decimal places (0.4472 vs 0.4473).

As a side finding, the non-associativity meter doubles as a conversation-stationarity signal, exhibiting monotonic decay at ≈ 2 percentage points per turn across four stationary-content sessions — a substrate-level geometric measurement that correlates with what Clark & Brennan (1991) call conversational grounding.

## What this paper is

A documentation of:

1. **The Davis Kähler generator** as a single mathematical object instantiated by two operationally distinct downstream consumers (a data-substrate consumer and a physical-substrate consumer).
2. **An eight-layer catalog** of geometric machinery on that generator, each layer with a closed-form prediction and a runtime validation.
3. **The optionality contract** as an independently citable engineering pattern: feature-gate plus typed-refusal vocabulary plus byte-invariance pinning in CI.
4. **Cross-team validation evidence** that the catalog's predictions hold quantitatively — to rounding precision on Marcella, to ≤ 3×10⁻⁴ rel.err. on the DPU.
5. **A side observation** that the L7.5 non-associativity meter doubles as a conversational-momentum signal, framed as a geometric measurement of the grounding state described in the Clark–Brennan dialogue-systems literature.

## What this paper is not

- A complete spoken-dialogue-systems paper (no ASR, intent-classification, or turn-taking machinery).
- A claim that the L7.5 +7.6 pp prediction generalizes to non-$S^{383}$ substrates with the same precision (population result is future work).
- A claim that the deep-trace single-session observation in §6.5 is a population result (it is a case study; n=1).
- A claim that the DPU validation is an out-of-sample prediction of the bilayer Berry phase (BLG has a known closed form; the DPU section is a cross-substrate consistency check on the catalog's L7.1 predicate).

## Reproducibility

The math validation reported in this paper is **fully reproducible from the public `gigi` repository alone** — no Marcella source, no DPU simulation-engine source, and no monorepo workspace required. The two consumers (Marcella, DPU) are the *targets* of the validation; the validation tests themselves live in the open repository.

**Repository:** <https://github.com/nurdymuny/gigi/>

Clone, then run from the repo root:

```
git clone https://github.com/nurdymuny/gigi.git
cd gigi
```

**Python validation** — covers L7.1 integrality on Wu–Yang $S^2$, McCann–Koshino analytic BLG, and an independent Python Wilson-loop discretization of the BLG 2-band Hamiltonian. From `gigi/theory/kahler_upgrade/validation/`:

```
python -X utf8 validation_tests_v5.py
```

3 tests, ~50 ms; captures `results_v5.txt` in the same directory.

**Rust validation** — exercises the production `src/geometry/line_bundle.rs::holonomy_debt` call on real consumer-facing data shapes. From the `gigi` repository root:

```
cargo test --release --test kahler_l7_real_data_smoke
```

Two tests: `real_sensor_data_l7_quantization_lifecycle` and `real_sensor_data_no_kahler_refuses_l7_surfaces`.

**What lives in private code (not required for reproducibility):**

- The DPU simulation engine's Wilson-loop implementation (Table 1 column "Wilson (Rust)") is part of the patent-protected DPU codebase (U.S. Provisional 64/073,981) and not part of public `gigi`. The relative-error agreement with the analytic formula in column "rel.err." is checked by the Python validation above against the same McCann–Koshino reference and provides an independent open-source confirmation of the implementation-consistency claim.
- The Marcella runtime that ran the 30-prompt A/B harness in §6.1–6.5 is not open-source; the A/B results are reported as a measurement performed on the live public Marcella service.

**Reproducible-build configuration** for the byte-identical no-feature build (Proposition 5.1), applied to the public `gigi` workspace:

- locked `Cargo.lock`
- `SOURCE_DATE_EPOCH` fixed across commits
- `RUSTFLAGS="--remap-path-prefix=$PWD=."`
- `-Ccodegen-units=1 -Cstrip=debuginfo`
- fixed toolchain pin (`rust-toolchain.toml`)

**Production source modules referenced in the paper** (all in public `gigi/src/`):
