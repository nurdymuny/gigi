# SPECTRAL_GAUGE BULK — the sparse interior arm is live

Hallie —

The BULK grammar is complete and the sparse interior solver is in. `SPECTRAL_GAUGE <bundle> ON FIBER (theta) GROUP U(1) MODE MAGNETIC BULK k [AROUND sigma] [IN [a,b]]` now serves three centerings: `BULK k` auto-centers on the positional median (the rank-`floor(V/2)` level — the number-variance object, robust to an asymmetric DOS, not the midrange); `AROUND sigma` returns the k eigenvalues nearest sigma; `IN [a,b]` returns every level in the closed interval (k as a safety clamp). BULK is magnetic-only — the interior window is defined on the complex-Hermitian magnetic spectrum, which is the RH object.

The routing is the piece you asked for. Below the dense ceiling (default 4096, opt-in 8192 via `GIGI_DENSE_CEIL`) BULK is a re-centering slice on the dense spectrum. PAST the ceiling — exactly where you need it, V = 13824 and 32768 — BULK no longer returns `SparseUnavailable`; it routes to a pure-Rust Chebyshev-filtered interior eigensolver (Option B, no new dependency). It assembles the magnetic Laplacian as a complex CSR (never a dense V×V matrix), brackets the spectrum with Gershgorin, builds a Jackson-damped Chebyshev bandpass around the target window, and runs block subspace iteration with full reorthogonalization + Rayleigh-Ritz (the same `nalgebra::SymmetricEigen` on the small projected block that the dense path already trusts). The magnetic-Laplacian convention is byte-identical to the dense path (`L[i,j] -= e^{i·theta}`, conjugate partner, unit degree on the diagonal), so sparse and dense agree where they overlap.

The response fields are the honest completeness surface you specifically asked for. Every BULK row carries `eigenvalues` (the window, ascending) and `mode_used`. On the sparse path `mode_used = "sparse_interior"` and the row adds `converged` (int), `max_residual` (f64), and `iterations`. `converged` is the number of returned pairs that passed the residual gate inside the target window; `max_residual` is the max over those pairs of `||L v - lambda v|| / ||v||`. The gate is pinned at `RES_TOL = 1e-8` — dense `SymmetricEigen` residuals run 1e-12..1e-14, so 1e-8 is a comfortable, honest certificate that each returned value is a TRUE eigenpair, not a filter artifact. When the oversampled subspace yields fewer than k converged pairs in the window, the arm grows the subspace and raises the filter degree (bounded retries); if it is still short it returns `converged < k` and does NOT claim k. It never silently reports a full window it did not certify.

Here is the honest completeness scope, stated plainly. For V <= 2048 I proved completeness against dense ground truth: dense-solve the FULL spectrum, sparse-interior-solve the center window, and assert the sparse window EXACTLY equals the dense center-k eigenvalues — same values (<= RES_TOL), same count, same multiplicities, no miss, no extra. This is anchor SP1, across three window sizes and both center positions (auto + off-center AROUND sigma), with the SP1 table in the ship report. The near-degenerate case (SP2) is the one that matters most for a filter: a ~1e-6 split pair (uniform-flux magnetic cycle) and an exactly-degenerate pair (two disjoint identical copies) — both members are returned, no merge. That is the classic completeness failure mode and it is fenced.

At V = 13824 and 32768 there is no dense ground truth — none exists, which is exactly why you asked for the converged/residual field. The guarantee there rests on three legs: (1) the residual gate — every returned pair is a certified eigenpair below RES_TOL; (2) `converged == k` — the count is complete; (3) the Jackson-damped Chebyshev filter's selectivity plus the oversample margin against near-degenerate merges. The honest caveat, in writing: a missed bulk level at V = 32768 cannot be POSITIVELY excluded by ground truth, because there is no ground truth to check against. How to gate on it: require `converged == k` AND `max_residual < 1e-8` on every window you keep; if `converged < k`, the arm is telling you it could not certify the full window — grow the subspace (oversample) or raise the filter degree and re-run, and if it stays short, treat that window as suspect rather than trusting a short count. That is the number the field exists to give you.
